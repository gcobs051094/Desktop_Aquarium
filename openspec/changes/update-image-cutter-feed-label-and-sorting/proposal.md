# Change: Update Image Cutter Feed Label and DFS Sorting Order

## Why
使用者需求變更：
1. **飼料選項標籤**：分類選單中的「飼料」選項需要改為「飼料 & 錢」，以更準確地反映該分類的用途
2. **儲存順序**：DFS 檢測後的區域儲存順序需要改為由左往右，而非原本的「由上而下、行內由左而右」的排序方式，以符合使用者的工作流程需求

## What Changes
- **更新分類選單標籤**：將分類依據下拉選單中的「飼料」選項改為「飼料 & 錢」
  - 僅修改顯示文字，內部對應的 `classify_basis` 值仍為 `"feed"`（索引 2）
  - 不影響現有功能邏輯
- **簡化 DFS 排序邏輯**：將區域排序改為純粹的由左往右（依左邊界 x 座標）
  - **檢測階段**（`detect_dfs_regions`）：移除複雜的「先分組到行、再行內排序」邏輯，改為直接依 `left` 座標排序
  - **裁切階段**（`start_dfs_cropping`）：
    - 自訂類別模式：使用由左往右排序的區域列表，類別內排序也改為由左往右
    - 一般輸出模式：使用由左往右排序的區域列表
    - 魚分類模式：仍需要「橫排」邏輯來對應行為資料夾，因此僅在該分支中保留行分組邏輯，但不影響其他模式的由左往右順序

## Impact
- **修改能力**：圖片處理工具（image-processing）
- **受影響的代碼**：
  - 修改 `tools/image_cutter_gui.py`：
    - 更新分類選單選項文字（第 822 行）
    - 簡化 `detect_dfs_regions` 中的排序邏輯（約第 1105-1155 行）
    - 更新 `start_dfs_cropping` 中的排序邏輯（約第 1415-1560 行）
- **受影響的目錄**：`tools/` 目錄
- **非 BREAKING**：向後兼容，現有功能不受影響，僅改變顯示文字和排序順序
