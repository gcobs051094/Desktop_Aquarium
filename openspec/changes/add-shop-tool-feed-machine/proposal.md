# Change: 商店新增「工具」Tab 與飼料投食機（Feed Machine）

## Why
目前商店僅提供「魚種 / 寵物 / 飼料」頁籤。為了擴充可互動與可視化的桌面裝置內容，需要在商店新增「工具」欄位，並加入「飼料投食機」做為第一個工具項目：達成解鎖條件後可在水族箱固定位置顯示素材，且可透過下拉選單調整顏色。

## What Changes
- **商店 UI**：`ShopOverlay` 新增一個「工具」tab（與既有 tab 一致為可捲動卡片列表）。
- **工具：飼料投食機**
  - **預設狀態**：未解鎖時以灰黑色為預覽/顯示，並顯示解鎖條件。
  - **解鎖條件**：獲得 10 隻成年孔雀魚（以專案既有的「曾達到最大數量」里程碑判定）。
    - 註：目前 `resource/fish/孔雀魚/` 無階段子資料夾，程式會以 `species="孔雀魚"` 且 `stage="small"` 建立魚；本變更將以「孔雀魚曾達到數量里程碑」視為「成年孔雀魚」條件的對應（避免引入孔雀魚成長階段的大改動）。
  - **解鎖後交互**：解鎖後原本的按鈕改為「顏色下拉式選單」，可選擇並即時套用投食機顏色（素材來源 `resource/feed_machine/投食機_<顏色>.png`），預設為灰黑色。
- **水族箱顯示**：解鎖後在水族箱座標 (25, 240) 顯示投食機圖片；顏色變更後即時更新顯示。
- **存檔/載入**：投食機的解鎖狀態與顏色選擇 SHALL 被保存並於啟動時恢復（向後相容：舊存檔缺欄位時使用預設值）。

## Impact
- **受影響能力**：水族箱使用者介面（aquarium-ui）、遊戲經濟（game-economy）、遊戲狀態管理（game-state）
- **受影響代碼（預期）**：
  - `aquarium_window.py`
    - `ShopOverlay`：新增「工具」tab 與投食機卡片、顏色下拉選單事件
    - `TransparentAquariumWindow`：解鎖條件判定、狀態保存/載入、將投食機顯示狀態傳入水族箱
    - `AquariumWidget.paintEvent()`：繪製投食機素材（固定座標）
  - `game_state.py`：預設 state 與缺欄位預設值（新增工具相關欄位）
  - （可選）`config.py`：新增 `TOOL_CONFIG` 或常數（顏色清單、預設顏色、座標）
