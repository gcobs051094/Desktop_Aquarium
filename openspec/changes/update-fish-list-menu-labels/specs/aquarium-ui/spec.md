## MODIFIED Requirements

### Requirement: 投放魚功能
系統 SHALL 提供「投放魚」按鈕，點擊後展開清單顯示所有可用的魚種，使用者選擇後在水族箱內新增一條該類型的魚。

#### Scenario: 魚種清單展開
- **WHEN** 使用者點擊「投放魚」按鈕
- **THEN** 按鈕下方展開一個清單選單
- **AND** 清單列舉 `resource/fish/` 目錄底下的**一層資料夾**（魚種目錄），不進入子目錄
- **AND** 清單項目顯示魚種資料夾名稱（例如：「guppy」、「puppy」）

#### Scenario: 新增魚類到水族箱（魚種目錄）
- **WHEN** 使用者從清單中選擇一個魚種（傳入魚種目錄路徑，如 `resource/fish/guppy`）
- **THEN** 系統在該魚種目錄下解析出名稱含 "small" 的子目錄作為載入路徑
- **AND** 載入該路徑的游泳與轉向動畫（使用 "5_吃飽游泳" 和 "7_吃飽轉向" 行為）
- **AND** 在水族箱內隨機位置新增一條該類型的魚
- **AND** 魚的初始方向為隨機角度（0-360度）
- **AND** 魚的游動速度為隨機值（0.4-0.8 像素/幀）

#### Scenario: 新增魚類到水族箱（階段目錄，向後相容）
- **WHEN** 系統收到階段目錄路徑（如 `resource/fish/guppy/small_guppy`）
- **THEN** 直接以該路徑載入動畫並新增魚，行為與既有邏輯一致
