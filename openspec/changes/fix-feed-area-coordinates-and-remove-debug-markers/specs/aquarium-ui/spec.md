# Spec: 修正投食機區域座標系統與滑鼠事件處理

## 概述
修正當 `FEED_MACHINE_AREA_WIDTH` 設定為 150 時，主視窗左側空白區域與水族箱區域的座標轉換問題，確保滑鼠事件正確處理，使水族箱左半邊可以正常投放飼料。

## 需求

### 1. 左側空白區域配置
- 主視窗左側應有可配置的空白區域用於放置投食機
- 空白區域寬度由 `FEED_MACHINE_AREA_WIDTH` 常數定義（預設 150 像素）
- 空白區域高度與主視窗高度一致

### 2. 區域布局
- 主視窗布局：`左側空白區域 | 水族箱區域 | 右側控制面板`
- 水族箱區域向右移動 `FEED_MACHINE_AREA_WIDTH` 像素
- 控制面板區域也向右移動相同距離
- 視窗大小 = `FEED_MACHINE_AREA_WIDTH + 水族箱寬度 + 控制面板寬度`

### 3. 滑鼠事件處理
- 水族箱區域的點擊應正確轉換為水族箱部件相對座標
- 左側空白區域的點擊應映射到水族箱左側區域（x 座標直接使用）
- 投食機部件的透明區域不應阻擋水族箱區域的點擊
- 投食機圖片的實際區域可以特殊處理（未來可能添加互動功能）

### 4. 座標轉換規則
- **水族箱區域點擊**：`aquarium_local_pos = pos - self.aquarium_rect.topLeft()`
- **左側空白區域點擊**：`aquarium_local_x = pos.x()`（直接映射）
- **投食機部件透明區域**：事件穿透到父視窗

### 5. 視窗遮罩
- 左側空白區域應包含在視窗遮罩中，可接收滑鼠事件
- 遮罩應動態更新以適應視窗大小變化

## 實作細節

### 滑鼠事件處理優先順序
1. 優先檢查水族箱區域（`aquarium_rect.contains(pos)`）
2. 其次檢查左側空白區域（`feed_machine_area_rect.contains(pos)`）
3. 最後檢查投食機部件圖片區域

### 投食機部件事件穿透
- 使用 `event.ignore()` 和 `QApplication.sendEvent()` 實現事件穿透
- 只有點擊在投食機圖片實際區域內才特殊處理
- 透明區域的事件應穿透到父視窗

## 測試要求
- 當 `FEED_MACHINE_AREA_WIDTH = 0` 時，功能應與之前一致
- 當 `FEED_MACHINE_AREA_WIDTH = 150` 時，水族箱左半邊應可正常投放飼料
- 左側空白區域的點擊應正確映射到水族箱
- 投食機圖片區域不應阻擋投放飼料
- 投食機透明區域不應阻擋投放飼料
