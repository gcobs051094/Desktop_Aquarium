# Change: 重構魚類吃飼料行為規則

## Why
目前的魚類吃飼料行為存在以下問題：
1. **轉向中碰到飼料會觸發吃動畫**：魚在轉向過程中碰到飼料時，會中斷轉向動畫並播放吃動畫，導致視覺上出現「頭部先轉回轉向前，然後再吃」的不自然現象
2. **吃動畫播放中仍可吃飼料**：魚在播放吃動畫時，如果碰到其他飼料，會立即觸發新的吃行為，導致動畫被中斷
3. **缺乏冷卻機制**：吃完飼料後沒有冷卻時間，魚可以立即追下一顆飼料，行為過於積極

需要重新定義「吃」的概念，將「行為」與「動畫」分離，並建立明確的規則來處理各種情況下的吃飼料邏輯。

## What Changes
- **分離「吃」的概念**：將「吃」分為「行為」（飼料消失、成長度增加、冷卻）與「動畫」（播放吃飼料動畫）
- **轉向中碰到飼料**：只觸發吃的行為（飼料消失、成長、冷卻），不播放吃動畫，不中斷轉向
- **吃動畫播放中碰到飼料**：不觸發任何吃的行為，飼料繼續存在並往下移動
- **冷卻機制**：觸發吃的行為後，魚必須等待 1 秒才能再次觸發吃的行為（此期間不追飼料）

## Impact
- Affected specs: fish-behavior（新增吃飼料行為規範）
- Affected code: `fish.py`, `aquarium_window.py`

## Implementation Details
1. **新增 `consume_feed()` 方法**：僅觸發吃的行為（計數、成長、冷卻），不播放動畫
2. **修改 `eat_feed()` 方法**：觸發吃的行為與動畫，動畫結束時才開始冷卻計時
3. **新增 `feed_cooldown_timer`**：追蹤冷卻時間，冷卻期間不追飼料
4. **修改碰撞檢測邏輯**：
   - 吃動畫播放中（`state == "eating"`）：跳過碰撞，飼料不消失
   - 冷卻期間（`feed_cooldown_timer > 0`）：跳過碰撞，不觸發吃的行為
   - 轉向中（`state == "turning"`）：呼叫 `consume_feed()`，只觸發行為
   - 游泳狀態：呼叫 `eat_feed()`，觸發行為與動畫
