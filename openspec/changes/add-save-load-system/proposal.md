# Change: 新增遊戲狀態儲存與載入系統

## Why
使用者需要能夠儲存遊戲進度，包括水族箱內的魚種、魚種數量、金額、解鎖魚種等狀態。當關閉程式後，下次啟動時應能恢復到關閉前的狀態。此外，程式將打包為 exe 提供給其他使用者，且會頻繁更新新的魚種、寵物、飼料、大便等內容，必須確保舊版本使用者可以直接替換 exe 檔案，且存檔格式向後相容，不會因新版本新增的內容而導致存檔損壞或無法載入。

## What Changes
- **遊戲狀態管理模組**：新增獨立的 `game_state.py` 模組，負責遊戲狀態的序列化、反序列化、儲存與載入。
- **儲存格式**：使用 JSON 格式儲存遊戲狀態，採用版本號機制與向後相容的資料結構設計。
- **儲存位置**：存檔檔案存放在使用者資料目錄（Windows: `%APPDATA%/Desktop_Feeding_Fish/save.json`），確保打包為 exe 後存檔不會與程式檔案混在一起。
- **自動儲存**：在關鍵狀態變更時自動儲存（魚類新增/移除、金額變更、解鎖狀態變更），並在程式關閉時執行最後一次儲存。
- **自動載入**：程式啟動時自動載入存檔，恢復遊戲狀態。
- **版本相容性**：存檔格式包含版本號，新版本能讀取舊版本存檔，並自動遷移或忽略未知欄位，確保向後相容。
- **解鎖狀態追蹤**：新增解鎖魚種的追蹤機制，記錄使用者曾經達到過的魚種數量里程碑。

## Impact
- **受影響能力**：遊戲狀態管理（game-state）、水族箱核心邏輯（aquarium-core）
- **受影響代碼**：
  - 新增 `game_state.py`：遊戲狀態管理模組
  - `aquarium_window.py`：整合儲存/載入功能，追蹤解鎖狀態
  - `fish.py`：可能需要支援從狀態重建魚類實例
  - `config.py`：可能需要定義存檔格式版本號常數
