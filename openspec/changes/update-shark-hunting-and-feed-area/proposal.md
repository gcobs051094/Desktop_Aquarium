# Change: 鯊魚主動追獵、死亡動畫調整與飼料投放區域限制

## Why
改善鯊魚行為使其更具掠食者特性（主動追獵幼鬥魚），調整死亡動畫觸發時機（升級與被鯊魚吃掉不觸發死亡動畫），以及限制飼料投放區域在魚可游動範圍內、修復商店飼料 tab 按鈕被裁切的 UI 問題。

## What Changes
- **鯊魚主動追幼鬥魚**：鯊魚在可進食狀態（上次吃魚後經過 300 秒或從未吃過）時，會主動朝最近的幼鬥魚移動追獵，而非僅被動等待碰撞。
- **被鯊魚吃掉不播死亡動畫**：被鯊魚吃掉的幼鬥魚直接從列表移除並消失，不觸發死亡動畫（灰階上移淡出）；鯊魚則觸發吃飯動畫。
- **魚升級不播死亡動畫**：魚升級時舊魚直接從列表移除，不觸發死亡動畫，新階段魚在同位置出現。
- **鬥魚碰孔雀魚加速擴展**：鬥魚碰觸孔雀魚時，除了追飼料速度 +0.2，平常游泳/轉向移動速度也同樣增加，維持 2 秒。
- **飼料投放區域限制**：可投放飼料的區域限制在魚能夠游動的範圍內（邊緣向內 50 像素），點擊邊緣區域不投放飼料。
- **商店飼料 tab 按鈕修復**：修正核廢料欄位按鈕被右側裁切的問題，調整卡片佈局讓文字能縮小換行、按鈕固定顯示。

## Impact
- **受影響能力**：魚類行為（fish-behavior）、水族箱 UI（aquarium-ui）
- **受影響代碼**：
  - `fish.py`：`update()` 新增 `prey` 參數供鯊魚追獵幼鬥魚；鬥魚孔雀魚 buff 擴展到平常移動
  - `aquarium_window.py`：
    - `update_fishes()` 傳入 prey 供鯊魚追獵
    - `_check_shark_eat_betta()` 回傳被吃魚，由呼叫端移除並觸發鯊魚吃飯動畫
    - `_on_fish_upgrade()` 舊魚直接移除不設死亡
    - `on_aquarium_clicked()` 限制飼料投放區域
    - `_update_feed_items()` 修正卡片佈局讓按鈕不被裁切
